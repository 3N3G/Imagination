## Mechanistic story (one paragraph)

Three probes agree on a single axis. Ordered by strength:

> C_grounded > B_thinking > A_top2M > A_full

1. **Held-out Δ(shuf−real) NLL** (loss-level content-reading): C +0.195 > B +0.085 > A_full +0.026.
2. **Direction-CF hidden-flip at step 0 / multistep**: B dominates step 0 (16.7%); C dominates multistep (36.8% at step 75).
3. **v2 die return drop** (behavior-level): C −4.90 > B −4.50 > A_top2M −2.28 > A_full −0.76.

At the loss level, the more fidelity in the training prediction ("C grounded with future context" > "B thinking with reasoning" > "A predonly fast-generate"), the more discriminatively the policy weights content. At the behavior level, the same ranking shows up as *how much a lie in the imagination tells on the policy*. Content-sensitivity is a single axis that fidelity training inflates, and v2 measures the clean vulnerability that this sensitivity creates.

The A_full vs A_top2M split within the same track is the dose-response: more training data → more content-integration buffer → less vulnerability to content-OOD. 10× data (A_full vs A_top2M, both predonly concise) buys ~1.5 return of robustness.

## Surprise finding — v1 numbers masked true content sensitivity

Going in, we expected v1 = content + format shift and v2 = content alone, so |Δ(v2)| ≤ |Δ(v1)|.

What actually happened for Track B:

| probe variant | B_top2M return |
|---|---|
| regular (thinking inference) | 16.31 |
| v1 die (thinking + "Instead of…" suffix) | 13.39 |
| **v2 die** (thinking template with algorithm replaced) | **11.26** |

v2 drops return further than v1. A consistent read: v1's "IMPORTANT OVERRIDE / Instead of / worst possible future" output-format signals are distinctive enough that the policy can partially treat them as "not normal imagination, discount", similar to how it discounts `constant`-mode canned text. With v2, Gemini produces a grammatical, single-sentence, state-understanding + prediction output in the exact voice of regular concise — but describing the player drowning themselves. The policy has no easy way to flag that as suspect and integrates it at face value.

Implication: **v1 robustness probes on augmented-trained policies were biased toward "looks robust" by the format escape hatch.** v2 removes that escape hatch. The more trustworthy number for "how much does bad-play content hurt this policy" is v2, not v1.

For Track A, v1 ≈ v2: confirms A is genuinely content-blind (no integration to penetrate). The v1 null result on A was the true result; we just couldn't confirm it until we ran v2 somewhere that moved.

## Open questions / remaining work

1. **Exploration axis**: none of the current probes test whether LLM-imagination can accelerate discovery of resource/ladder/unseen-floor events. Worth considering a designed exploration probe (e.g. force imagination to suggest a specific floor-descend plan, measure whether policy acts on it and whether it expands reachable state coverage).
2. **Verification axis**: nothing in place that checks the policy's reliance on imagination via a consistency / self-check mechanism. Possible probe: inject *contradictory* imagination vectors across consecutive steps and measure action variance — does the policy tolerate contradiction or commit?
3. **A_top2M probes missing**: we have A_full and A_top2M v2 die/adv, but direction-CF / HP-perturb only ran for A_full. Fair-size A comparison (top-2M everywhere) is incomplete. If we want a matched-size axis of {predonly, thinking, grounded} all at 2M, we're still missing A_top2M on direction-CF and HP-perturb.
4. **Held-out Δ_shuf for A_top2M not measured**. Quick to run (5 min).
5. **v2 probe could extend** to "constant-embed analogue" (hidden = average v2-die embedding across samples) to test whether the return drop is specific to *dynamic per-step bad content* or just to *static off-distribution hidden*. Currently constant-embed uses the canned "Nothing happens. The player stands still…" text, which is closer to regular than to v2.
6. **C_grounded's wrong-sign HP-food response** is still the single most interpretable bug we've found. v2 result (C_grounded loses 5 return to plausible death content) *strengthens* the train/eval mismatch diagnosis — C specifically reads content as if it were oracle-quality predictions, so a plausible death description gets the same "this is what happens, plan around it" treatment that an oracle death label would warrant.
