# 2026-04-14

## Experiment 10 — Hidden-only baseline (no obs, pure AWR)

**Motivation:** Test whether the Qwen3 imagination embedding alone carries enough signal to drive a competent policy. Anchors the signal/noise question: if the embedding is rich enough, a policy that sees *only* it should do something. If not, this sets a floor.

**Model:** `ActorCriticHiddenOnly` — 3-layer MLP over hidden_state only, LayerNorm + Dropout, 5.27M params. No obs branch, no gating.

**Training:** pure AWR (`--oracle-loss-weight 0.0`, `--oracle-fraction 0.05`), LR 3e-4, entropy 0.01, 100K steps from scratch.

### Held-out validation (807,095 samples, train-distribution)

| Condition | NLL | Acc |
|---|---:|---:|
| real | 2.44 | 23.21% |
| zero | 3.42 | 12.63% |
| shuffled | 4.46 | 7.99% |

Clear `real > zero > shuffled` ordering; model *does* use embedding content on-distribution. But 23% peak accuracy is weak compared to unaug's obs-driven accuracy levels.

### Golden validation (38,108 samples, oracle Gemini trajectories)

| Condition | NLL | Acc |
|---|---:|---:|
| real | 2.81 | 14.52% |
| zero | 2.60 | 14.49% |
| shuffled | 2.98 | 14.31% |

**Essentially flat.** Embedding adds *nothing* on the golden distribution — zero is marginally better than real. The content signal that exists in train-distribution embeddings does not transfer to oracle-distribution embeddings.

### Online eval (50 eps, live Gemini + Qwen)

- **Return: 1.98 ± 1.01** (min 0.10, max 5.10)
- Length: 253 ± 102
- Achievements: 2.9 ± 1.0 (collect_sapling 49/50, collect_wood 48/50, collect_drink 20/50)

**vs unaug ActorCritic (obs only): 18.38** — catastrophic collapse (-16.4 pts).
**vs awr_aug_debug (obs + hidden, pure AWR): 16.30** — the -14.3 delta isolates the obs pathway's contribution.

### Verdict

- **The imagination embedding cannot substitute for obs.** Without observation input the policy collects basic resources but fails to survive or progress.
- **Train-distribution signal ≠ oracle-distribution signal.** The held-out real>zero>shuffled gap exists but evaporates on golden data, suggesting the embedding pathway fits train-set idiosyncrasies (maybe action prior via oracle-distribution labels) rather than transferable planning content.
- **Anchors the signal/noise debate.** This is direct evidence that the embedding itself, before any policy architecture choices, is too impoverished to drive competent behavior. Matches the direction-match pattern: the policy reads the embedding, but what's encoded is action-priors more than state-relevant plans.

## Experiment 11 — history-5 pipeline

**Pipeline:** Gemini+Qwen labelling with the 5-step-history prompt variant → merge → pure AWR 100K → val + 50ep live eval.

**Training:** Finished in 22 min after embedding (~12h40m total). Final model `/data/group_data/rl/geney/checkpoints/awr_hist5/final.pth` (14.27M params, ActorCriticAug width 512). Loss curves clean (awr loss ~2.1 by end, expl_var 0.98). First eval attempt (7091030) OOM'd at 256G loading 32 val files; resubmitted eval-only (7108241) with `--max-files 8`.

### Validation (ablation)

| | real NLL | real Acc | zero NLL | zero Acc | shuf NLL | shuf Acc |
|---|---:|---:|---:|---:|---:|---:|
| Held-out (807K) | 2.40 | 39.83% | 2.41 | 39.73% | 2.40 | 39.79% |
| Golden (38K) | 5.87 | 9.22% | 5.78 | 9.60% | 5.90 | 9.38% |

**Completely flat — the embedding is being ignored.** On both held-out and golden, real ≈ zero ≈ shuffled to within noise. The held-out accuracy (39.8%) is actually high, but it's driven entirely by the obs pathway. Golden NLL 5.87 is dramatically worse than for any prior embedding (unaug golden ≈ 2.60), suggesting the hist5 policy also doesn't transfer well to the oracle obs distribution.

### Online eval (still in flight at journaling time, ep 28/50)

- Returns so far: **18.69 mean** (range 14–23, n=27). Many eps 19–20. Episode 27 collected 22 achievements.

**Pattern matches a pure obs-only policy.** Return is in the same ballpark as unaug 18.38 — the added history context didn't degrade it, but the validation tables say the embedding contributes nothing, so any gain (if any) is a training-dynamics effect, not new information from the prompt.

### Verdict

- 5-step-history prompts generate embeddings the policy cannot (or chooses not to) use.
- Adds another data point to the signal/noise debate: making the prompt more informative from a human standpoint does *not* translate to more usable signal for this policy architecture. Consistent with the hidden-only collapse (Exp 10): the embeddings just aren't carrying planning-relevant content in a form the downstream policy can exploit.

## Experiment 12 — Gemini plays Craftax directly (pilot)

**Motivation:** Can Gemini clear a nontrivial bar (≥5 return) when wired
into the game as the actor, given the user-authored survive→ladder→
upgrade→explore algorithm as system prompt and a short action-history?
Anchors the signal-source question from the other side: if Gemini itself
can't play well with an explicit playbook, asking a small policy to learn
from its embeddings is harder than we thought.

**Setup:** Every env step, render symbolic obs → `obs_to_text` →
`filter_text_obs` → prepend the user's algorithm + last-5 action history →
call `gemini-2.5-flash`, parse `Action: <id>` from reply. Model is asked
for ≤2 lines, last line `Action: <id>`.

### Pilot v1 (script as first drafted) — 5 eps, 400 max steps

- Return **1.30 ± 0.75** (min 0.10, max 2.10).
- **Parse failure 83%** (1202/1440 Gemini calls).
- Root cause: I set `use_thinking=False` to *allow* thinking, but the
  existing `call_gemini` wrapper's flag is inverted (`True` means
  "override default to budget=0 i.e. disable thinking"). So thinking was
  enabled *with default budget*, and the 512-token output budget was
  consumed before the model emitted `Action:`. Default-on-parse-fail was
  NOOP → apparent score is just a random-NOOP baseline.

### Pilot v2 — 5 eps, 400 max steps

- Return **4.06 ± 1.98** (min 0.10, max 5.10).
- 4/5 eps hit 5.0–5.1; 1/5 died at step 338 (drink→0, energy→0, food→2:
  kept moving UP while stats crashed — "Survive" priority wasn't honored).
- **0 parse failures / 0 API failures** across 1,628 calls.
- Mean latency 0.77s/call (thinking off), episode wall 200–293s each.

Changes vs v1: (1) explicit `use_thinking=True` → thinkingBudget=0
(disables reasoning so the full 256-token budget goes to the answer),
(2) prompt says "Respond in ≤2 lines, last line: Action: <id>",
(3) permissive parser (accepts `Action: N`, `Action N`, trailing number).

### Verdict

- **5+ return achievable** with a structured algorithm + a few steps of
  history. The pilot clears the user's bar (5+, on 4/5 runs); the 18+
  regime is nowhere near, but this is a first-try minimal prompt.
- **Failure mode is "SURVIVE" not being honored**, not parsing or
  reasoning. Gemini picked directional moves while stats bottomed out.
  Next iteration should either (a) front-load "STOP and drink/eat if any
  stat ≤3" at the top of every prompt, (b) pre-gate the algorithm so step
  1 is always "is any stat critical?".
- **Gemini-plays-well is non-trivial but not strong yet.** Return 4–5 is
  at the floor of what an obs-only learned policy gets (unaug 18.4). The
  gap between "Gemini actor" and "tiny policy trained on obs" remains
  large, which reframes the signal/noise debate: the data-generating
  Gemini itself isn't producing planning-competent signal, so distilling
  it into embeddings for a policy was upper-bounded from the start.
