## Key Findings from Accuracy Grid

### 1. AWR no aug (row 1): model ignores embeddings entirely
Training accuracy: zero (39.9%) > real (34.7%) = shuffled (34.7%). Since the model was trained with zero embeddings, zero is the in-distribution condition. Real = shuffled confirms the model treats all non-zero embeddings identically — the embedding branch is completely inactive.

Oracle accuracy: zero (9.8%) > shuffled (6.0%) ≈ real (5.9%). Even the "best" oracle accuracy is only 9.8% — the model has no oracle-specific information. Random baseline would be 1/17 ≈ 5.9%.

### 2. AWR aug (row 0): embeddings useful for training distribution, OOD on oracle
Training accuracy: real (44.4%) > zero (38.5%) > shuffled (33.3%). The model is using Qwen embeddings semantically for PPO trajectories — real embeddings give +5.9pp over zero, +11pp over shuffled. The embedding branch is genuinely active.

Oracle accuracy: zero (14.8%) > shuffled (14.0%) > real (13.5%). On oracle states, real embeddings are *slightly harmful* — oracle game states produce OOD Qwen embeddings that don't match the model's learned embedding-to-action mapping. This explains why this model doesn't collapse catastrophically: it can't easily memorize oracle states since the embedding is confusing rather than helpful for that distribution.

### 3. BC models (rows 2-5): 92% oracle accuracy — complete memorization
All four BC models achieve ~92% oracle top-1 accuracy using real embeddings, vs 5.9% random baseline. The models have perfectly memorized the 38K oracle samples.

**Hidden state is the primary discriminating signal, not the embedding:**
- AWR+BC noaug (row 2): shuffled oracle acc = 90.3% (nearly as good as real 92.2%). The model identifies oracle samples primarily from PPO hidden states (LSTM states), not Qwen embeddings.
- AWR+BC aug (rows 3-5): shuffled oracle acc = 88-90%. Same pattern — hidden states discriminate oracle samples, embedding adds ~3-4pp.

**Embedding adds extra discriminability for aug models:**
- Zero embedding on oracle: 37.6% for noaug vs 46.8-49.8% for aug models. The aug model's BC head has an additional oracle-specific embedding branch that provides extra signal even under zero-emb evaluation.

### 4. BC models on training data: NLL is catastrophically high
AWR+BC noaug (row 2): training real NLL = 5.41, vs 2.63 for pure AWR noaug. The BC objective has *degraded* PPO action prediction ability — the model has partially overwritten its PPO policy to output oracle actions. This quantitatively confirms the performance collapse.

AWR+BC aug (rows 3-5): training real NLL = 2.05 (comparable to AWR aug's 1.93). The real embedding pathway partially preserves PPO action prediction because the model learns to use the embedding to switch modes — but at the cost of producing oracle-specific behavior when encountering oracle-like embeddings.

### 5. Why BC causes catastrophic collapse
The accuracy grid tells a complete mechanistic story:
1. Oracle training data has highly stereotyped LSTM hidden states (human trajectory structure)
2. BC loss teaches the model to reproduce oracle actions conditioned on these states
3. At eval time, any new game state that triggers oracle-like hidden states causes the model to output oracle actions (which are OOD for the current game state)
4. With real embeddings, the model gets an *additional* oracle tag, making the mode-switching sharper and the collapse worse
