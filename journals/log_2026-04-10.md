## Key Findings from Accuracy Grid

Architecture note: `hidden_state` in all data files = 4096-dim Qwen embeddings of Gemini future-state predictions. No recurrent state anywhere. The three conditions in validate_awr test: `real` (actual normalized Qwen embedding), `zero` (all-zeros), `shuffled` (Qwen embeddings permuted across samples within the batch).

### 1. AWR no aug (row 1): ignores embeddings by construction
Trained with `hidden_mode=zero` — Qwen embeddings were always zeroed during training. So zero is the in-distribution condition. At validation: real (34.7%) = shuffled (34.7%) < zero (39.9%) — confirms the embedding branch carries no trained signal; any non-zero input hurts. Oracle accuracy: zero best (9.8%), real/shuffled both ~5.9% (near random). This is not a surprising result, it is the trivial expected behavior.

### 2. AWR aug (row 0): Qwen embeddings are genuinely used on training distribution
Trained with real Qwen embeddings. Training accuracy: real (44.4%) > zero (38.5%) > shuffled (33.3%). The model uses embedding content — wrong embeddings (shuffled) are worse than no embedding (zero), showing semantic sensitivity. Meaningful +5.9pp over zero, +11pp over shuffled.

Oracle data: zero (14.8%) > shuffled (14.0%) > real (13.5%). Oracle game states (human-played) produce structurally different Gemini labels → different Qwen embeddings → slightly OOD for the training-distribution model. Real oracle embeddings marginally confuse the model. This may partly explain why AWR aug (no BC) doesn't catastrophically collapse: the oracle Qwen embeddings don't map cleanly to the model's learned embedding→action function.

### 3. BC models (rows 2-5): 92% oracle accuracy — observation is the primary discriminating signal
All BC models achieve ~92% oracle accuracy with real embeddings. But the crucial comparison is shuffled:
- AWR+BC noaug (row 2): shuffled oracle acc = **90.3%** (vs real 92.2%)
- AWR+BC aug (rows 3-5): shuffled oracle acc = **88-90%** (vs real 92%)

Shuffled embeddings within the oracle evaluation batch are still oracle-distribution Qwen embeddings (just mapped to different timesteps). The near-parity between real and shuffled tells us the model is **not** using the semantic content of a specific embedding to predict that sample's action — it is using the **observation** (game state features). Human-played game states are structurally different from PPO-played states (different item inventories, health, map progression), and BC has trained the model to reproduce oracle actions for oracle-like observations. The Qwen embedding adds ~3pp of additional signal on top.

Zero embedding on oracle: 37.6% (noaug) vs 46.8-49.8% (aug models). The aug models get extra oracle identification signal from the embedding distribution even under zero-emb eval, because the BC head's embedding branch has been tuned on oracle data.

### 4. BC models on training data: PPO action prediction is degraded
AWR+BC noaug (row 2): training NLL = 5.41 vs 2.63 for pure AWR noaug. BC has partially overwritten the PPO policy — the model predicts oracle-mode actions even on training-distribution states, explaining the performance collapse.

AWR+BC aug (rows 3-5): training real NLL = 2.05 (comparable to AWR aug 1.93). The model partially preserves PPO behavior because it can use the Qwen embedding to mode-switch — PPO states get training-distribution embeddings, oracle states get oracle-distribution embeddings. But at eval time, generating novel Qwen embeddings for new game states can trigger partial oracle-mode behavior.

### 5. Mechanistic story
1. BC loss on oracle (human) data teaches the model to reproduce human actions for human game states
2. Human game states are systematically OOD vs PPO game states — the model learns an observation-level oracle detector
3. At eval, when the agent encounters game states that pattern-match oracle observations (even partially), the model shifts toward oracle actions that are inappropriate for the current context
4. Real Qwen embeddings add a secondary oracle-detection channel — oracle game states generate different Gemini predictions → different Qwen embeddings → embedding-level oracle tag. This is why BC+aug collapses harder than BC+noaug (7.5 vs 12.3)

**Hypothesis:** Purpose-built embedding models (Qwen3-Embedding, Gemini embed) may capture Gemini text semantics better than generative hidden states, potentially pushing AWR aug above the no-emb baseline.
