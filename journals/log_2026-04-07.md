
# 2026-04-07: BC+AWR Failure Analysis & New Experiments

## The Core Problem

BC+AWR models get **2-4 return** vs **17-19** for pure AWR or unaugmented baselines, despite training on the same data and architecture fixes.

**control_v2_zero proves the V2 architecture works fine (18.40 return). The BC objective (imitating golden data) is the sole cause of collapse.**

### Data distributions
| Dataset | Episodes | Mean Return | Transitions | Role |
|---------|----------|------------|-------------|------|
| PPO training data | ~1192 eps/shard | 20.9 | 12.7M | AWR source |
| Golden (human play) data | 24 eps | 46.8 | 38K | BC source |
| Held-out test | 1 ep | 55.4 | 1,175 | Validation |

### Original unaugmented model on held-out test: 11.8% accuracy
**The unaugmented model that gets 19.1 return has only 11.8% accuracy on golden trajectory actions.** BC+AWR models get ~70% accuracy but only 2-4 return. High golden-action accuracy does NOT predict good gameplay.

## Bugs Found and Fixed in `train_awr_weighted_v2.py`

1. **Golden critic loss unweighted** (CRITICAL): `total_critic_loss = critic_loss + oracle_critic_loss` → fixed to `critic_loss + oracle_loss_weight * oracle_critic_loss`. Biased value function toward golden data's higher return distribution.
2. **Golden entropy leak**: `entropy_loss = -(ent_train + ent_oracle) * 0.5` → fixed to `entropy_loss = -entropy_train`. Leaked golden-data gradients even when oracle_weight=0.
3. **Shard size mismatch**: `MAX_DATASET_GB = 30` → changed to 60. Old value caused 21 shards with ~4.7K steps each vs unaugmented's 6 shards with ~16.7K steps each.

## Diagnostic Run Results (v9, instrumented)

### Hidden-source separability (FACT)
- After z-normalization: training hidden norm = 63.2, golden hidden norm = 101.9
- Mean cosine between source centroids: -0.23
- Within-golden cosine: 0.27 (correlated), within-training: 0.003 (random)
- **Guess**: The norm difference exists because the human player visits fundamentally different game states (deeper floors, more items, different strategies) than the PPO agent, producing different Gemini text → different Qwen embeddings. Raw norms are only 3% different (447 vs 460), but z-normalization amplifies per-dimension differences. Both datasets were labelled with the identical predict-state-only Gemini+Qwen pipeline — the difference is purely from game state distribution.

### Norm-as-tag test (FACT): Model does NOT use norm as source tag
Direct test on v8 checkpoint:
- `rescaled_to_train_norm` (same direction, 63.7 norm): **100% agreement** with real (104.4 norm)
- `random_dir_train_norm` (random, 63.7): 60.6% agreement
- `random_dir_golden_norm` (random, 104.4): 59.6% agreement
- Rescaling norm has zero effect on predictions. Random directions hurt equally regardless of norm.

### Counterfactual validation (FACT)
At step 7500 (near peak Δ):
- KL(real||zero) = 2.32 — large difference
- KL(real||shuffled) = 0.046 — tiny difference
- KL(real||mean_emb) = 0.027 — tiny difference
- agree(shuffled) = 92.4%, agree(mean_emb) = 94.6%
- **The model uses the hidden branch as a nonzero-vs-zero detector, not for per-sample semantic content.**

Pattern holds throughout training (2.5K-45K steps): KL(real||shuffled) stays ~100x smaller than KL(real||zero).

### Gradient conflict (FACT)
- grad_cos(AWR, BC) oscillates between -0.15 and +0.19, centering near 0
- AWR and BC gradient norms are comparable (ratio ~0.6-1.8)
- **Gradients are approximately orthogonal** — not directly opposing, but not cooperating

### Golden data memorization (FACT)
- BC loss on golden data: 1.41 → 0.03 by step 26K (50x collapse)
- Golden entropy: 1.03 → 0.08 (near-deterministic on golden samples)
- AWR loss: 2.85 → 2.2 (slow, steady decline, NOT memorized — 12.7M samples too large)
- The 38K golden samples are memorized within 20K steps. After that, BC loss provides no learning signal.

### Δ(real-zero) degradation (FACT)
- Peaks at +0.48 (step 7500)
- Degrades to ~0.30 by step 40K
- Consistent with model learning to use hidden as generic cue early, then losing discrimination as training continues

## Interpretation (GUESS)

The model is probably not using the sample-specific semantic content of imagination. It uses the hidden vector as a generic nonzero side input / distribution cue. Evidence:
- Shuffling embeddings barely changes behavior (92%+ action agreement)
- Mean embedding produces 94%+ agreement with real
- Zero is the only condition that substantially changes policy
- Adversarial/die embeddings don't hurt gameplay (~same return as gemini)

**Guess**: The BC objective memorizes the golden dataset very quickly (38K << 12.7M), then the model learns that "when hidden vector looks like the golden distribution → produce golden-style actions." But the golden actions come from a skilled human player whose decision-making the model can't replicate from the embedding alone. The hidden branch encodes "this is golden data" not "this is the semantic content of the imagination." Note: both datasets use the identical predict-state-only labelling pipeline — the embedding difference is purely from different game states visited by human vs PPO agent.

**Guess**: An alternative explanation is that the BC+AWR mixture objective itself creates a degenerate equilibrium where the hidden branch settles into a weak/non-semantic role because that's the easiest way to minimize both losses simultaneously.

## Architecture Changes

### ActorCriticAugV2 (added Apr 6)
- Obs branch: 3 layers (matching unaugmented)
- Hidden branch: 2 layers, additive injection
- Zero-init on hidden output layer
- Result: same ~2-3 return as V1, architecture wasn't the issue

### ActorCriticAugGated (added Apr 7)
- Same as V2 + learned scalar gate: `g = sigmoid(fc(obs_representation))`
- `combined = obs_out + g * hidden_out`
- Gate conditioned on obs only (detached), initialized near 0 (bias=-2)
- Model learns WHEN to use imagination

### Bug fixes applied to train_awr_weighted_v2.py
1. **Shard wraparound stats**: `_stats_frozen` flag prevents double-counting on cycle
2. **Gradient clipping**: `--max-grad-norm` with `clip_grad_norm_`
3. **Fresh diagnostic batches**: gradient conflict uses new samples, not stale training tensors
4. **Both-stream entropy**: opt-in `--entropy-both-streams` (default: training-only)
5. **Pre-scan hidden stats**: reads all files before loading first shard for global stats
6. **Config mutation**: mutable values flow through `args`/`device`, Config is immutable defaults

| Config | Grad Clip | Prescan | Both Ent | Return | Δ(real-zero) |

**Findings:**
- Grad clipping at 1.0 is the most helpful individual fix (+1.4 return vs no_clip)

- clip=5.0 is worse than no clip — 1.0 specifically works, not clipping in general
- Combined fixes (3.70) > old behavior (2.60), but still far from pure AWR (15-18)

## Conclusions (Apr 7)
- Golden BC is toxic due to OOD data (Floor 1-2 vs Floor 0), not the BC loss formulation
- In-distribution BC is harmless but adds nothing — model can't distinguish same-distribution embeddings
- Oracle labels teach semantic embedding use but hurt eval (train/eval label mismatch)
- PSF labels match eval but model ignores embeddings entirely
- **Open problem:** no setup both teaches semantic embedding use AND produces good gameplay
- **New direction:** History-conditioned prediction may bridge the gap — grounding Gemini in policy behavior without leaking future info
