# 2026-04-11: Freeze Experiment + V2 Architecture Comparison

## Motivation

BC is toxic because it rewrites the obs pathway to produce oracle-style actions for oracle-like observations, while the hidden branch becomes a nonzero-vs-zero source detector (Apr 7-10 findings). Direct intervention: **freeze the obs pathway** during BC so BC can only express learning through the imagination/hidden branch.

## Experiment 1: Freeze BC (4 configs)

**Architecture**: `ActorCriticAugLN` (512-width, 14.3M params total).

```
ao = tanh(LN(obs_fc1(obs)))              # pre-merge obs (1 layer)
ah = tanh(LN(hidden_fc1(hidden)))         # pre-merge hidden (1 layer)
ax = tanh(LN(fc1(cat([ao, ah]))))         # merge: concat 2W → W
ax = tanh(LN(fc2(ax)))                    # post-merge
out = actor_out(ax)                       # output head
```

**Pretrained checkpoint**: `awr_aug_debug/final.pth` — pure AWR (no BC), real embeddings, 100K steps, 16.30 ± 3.61 return (50 episodes). Trained on oracle-mode Gemini labels.

**Freeze modes**:
- `obs_branch`: freeze `obs_fc1 + ln1` (actor & critic). Trainable: hidden_fc1 + ln2 + fc1 + ln3 + fc2 + ln4 + out.
- `obs_and_post_merge`: freeze obs_fc1 + ln1 + fc1 + ln3 + fc2 + ln4 + out (actor & critic). Trainable: only `hidden_fc1 + ln2`.

Note: `obs_and_post_merge` works with concat merge because `fc1(cat([frozen_ao, trainable_ah]))` decomposes as `W[:,:512]·ao + W[:,512:]·ah + b` — the frozen left half processes obs, the frozen right half processes hidden. BC can only steer through `ah`.

**Data**:
- Training: `new_craftax_llm_labelled_results_shards/final_trajectories/` (126 files, 12.7M samples). Oracle-mode Gemini 2.5 Flash labels (sees future 15 steps) → Qwen3-8B layer-30 embeddings (4096-dim).
- Oracle/BC: `oracle_pipeline/final_trajectories/trajectories_000000.npz` (38K samples, 24 human episodes). Same pipeline: oracle-mode Gemini → Qwen3-8B embedding.
- Both datasets labelled identically. Embedding distributional differences arise from different game states (human vs PPO agent), not different pipelines.

| 0 | freeze_obs_bc | obs_branch | off (BC only) | hidden + merge + output | 7059343_0 |

**Hyperparams** (all 4): lr=1e-4, entropy=0.01, max_grad_norm=1.0, 50K steps, batch=256. BC-only: oracle_loss_weight=1.0. BC+AWR: oracle_fraction=0.05, oracle_loss_weight=0.5.

**Eval**: job 7059344 (afterok:7059343), 50 episodes each with `eval_online.py` (live Gemini+Qwen).

**Hypothesis**: Freezing obs pathway prevents BC from corrupting the learned observation representation. obs_and_post_merge is the stronger test — BC can only add corrections through the hidden projection into a frozen representation space. If return stays near 16 (AWR baseline), frozen obs protects the policy. If return drops to 7 (unfrozen BC level), freezing doesn't help because the hidden branch itself is the corruption vector.

## Experiment 2: V2 Architecture Comparison (2 configs)

**Architecture**: `ActorCriticAugV2` (512-width). Differs from ActorCriticAugLN:
- 3-layer obs branch (matches unaugmented ActorCritic depth) vs 1-layer
- 2-layer hidden branch with zero-initialized output (starts as obs-only) vs 1-layer
- Additive merge (`ao + ah`) vs concatenation (`cat([ao, ah]) → fc1`)
- 1 post-merge layer vs 2

**Data**: Same as Experiment 1 (oracle-mode Gemini labels, same training/oracle paths).

| ID | Tag | Config | Comparison baseline (LN arch) | Job |
|----|-----|--------|------------------------------|-----|
| 0 | v2_awr_aug | AWR only (no BC), real emb | awr_aug_debug: 16.30 ± 3.61 | 7059349_0 |
| 1 | v2_awr_bc_aug | AWR + BC (OF=0.05, OW=0.5), drop=0.2, wd=1e-3, ent=0.03 | awr_bc_aug_debug: 7.46 ± 4.91 | 7059349_1 |

**Hyperparams**: lr=3e-4, beta=10, 100K steps, batch=256.

**Eval**: job 7059354 (afterok:7059349), 50 episodes.

**Purpose**: Determine whether V2's deeper obs branch + additive merge + zero-init hidden branch naturally handles BC better or worse than ActorCriticAugLN. If V2 AWR baseline matches LN (~16-18), the architecture is interchangeable. If V2 BC+AWR differs substantially from LN (7.46), the merge mechanism matters.

## Code Changes

### `offline_rl/train_awr_weighted_v2.py`
- Added `--pretrained-checkpoint`: loads a .pth checkpoint before training
- Added `--freeze-mode {none, obs_branch, obs_and_post_merge}`: freezes layers via `requires_grad=False`. Supports both ActorCriticAugLN and ActorCriticAugV2/Gated.
- Added `--no-awr`: disables AWR loss entirely; BC-only training on oracle data. Full batch goes to oracle.
- Added `apply_freeze()` function: maps freeze modes to specific layer names per architecture
- Added `train_step_bc_only()` function: BC loss only (no AWR computation)
- Optimizer now uses only `[p for p in model.parameters() if p.requires_grad]`
- Shard rotation and gradient conflict diagnostics skipped in `--no-awr` mode

### New SLURM scripts
- `slurm/jobs/freeze_bc_run.sh`: array job for 4 freeze configs
- `slurm/jobs/eval_freeze_run.sh`: array eval job for freeze checkpoints
- `slurm/jobs/v2_compare_run.sh`: array job for V2 architecture comparison
- `slurm/jobs/eval_v2_compare_run.sh`: array eval job for V2 checkpoints
