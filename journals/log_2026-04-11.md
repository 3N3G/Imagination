## Infrastructure Fixes

### 1. embed.py: Incremental per-file saving

**Bug:** `pipeline/embed.py` collected ALL texts from ALL files into memory, ran ALL
embeddings, then saved. A job timeout mid-embedding lost everything.

**Fix:** Model-based backends (hf, qwen3_embed) now load the model once and process
files one at a time, saving each file immediately after embedding. The Gemini API
backend (gemini_embed) similarly processes per-file. Existing files are skipped on
resubmission.

### 2. rl partition nodes have no internet access

**Finding:** babel-v5-16 (rl partition) cannot reach the Gemini API. API calls hung
silently — no errors, no progress. The original `emb_cmp_gemini` job ran there for
20+ minutes with zero completions.

**Fix:** Gemini API embed phase must run on a CPU partition node (general/cpu has
internet). Split the emb_cmp_gemini job into:
- Embed phase: CPU partition, no GPU, internet access
- Train/eval phase: rl partition, GPU required
