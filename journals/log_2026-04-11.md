# 2026-04-11: Embedding Comparison Experiment Launched

## State of Play

BC+AWR is closed. The Apr 10 accuracy grid confirmed the mechanistic story: BC
memorizes oracle observations directly, embedding pathway is an amplifier not a cause.
Pure AWR (no BC) is the current training approach.

Current ceilings:
- No-aug pure AWR: **18.38 ± 2.69** (50-ep)
- Aug pure AWR (oracle labels, Qwen3-gen): **16.30 ± 3.61** (50-ep) — real embeddings
  slightly hurt vs no embeddings, which is consistent with oracle labels being OOD at
  eval time
- PSF labels (flash-lite, Qwen3-gen): **17.10** (10-ep, Apr 7) — matches eval distribution
  better, embeddings essentially ignored (real ≈ zero ≈ shuffled in accuracy grid)

The open question: **can we get a meaningful policy improvement from imagination at all?**
Two angles running in parallel:

---

## Experiment 1: Embedding Comparison

**Question:** Qwen3-8B is a generative model — its layer-30 hidden states were never
trained to produce useful text representations. Does a model explicitly trained for
embedding produce richer representations that actually help the policy?

**Three conditions** (same flash-lite PSF labels, same training setup, pure AWR):

| Condition | Model | Dim | Status |
|-----------|-------|-----|--------|
| Baseline (existing) | Qwen3-8B generative, layer-30 mean-pool | 4096 | done — **17.10** |
| Gemini Embed | gemini-embedding-001 API | 3072 | embedding running |
| Qwen3-Embed | Qwen3-Embedding-8B, last-token pool | 4096 | pending GPU slot |

If either dedicated embedding model beats 17.10 (and especially if real > zero in the
accuracy grid), it would suggest the embedding quality was the bottleneck all along, not
the imagination approach itself.

If neither beats 17.10, the embedding method is not the bottleneck — the labels
themselves (flash-lite PSF predictions) may be too generic to encode useful signal.

---

## Experiment 2: History-Conditioned Imagination (hist5)

**Question:** Predict-only labels are a snapshot — Gemini sees one state and guesses
the future without knowing what the agent has been doing. Passing the last 5 states
as history should ground predictions in actual policy behavior. Does richer context
produce embeddings the policy can actually use?

**Setup:** Same pure AWR training, same Qwen3-8B embedding, but labels were generated
with the agent's last 5 (obs, action, reward) tuples prepended to the Gemini prompt.
Labels cover all 158 filtered_trajectories files (~4.27M calls, ~$938, already complete).

Currently running: embedding phase (4.27M texts → Qwen3-8B), then merge → train →
validate → eval 50ep. ETA ~2 days.

If hist5 beats 17.10 and shows real > zero in the accuracy grid, history context is
the missing ingredient. If it still shows real ≈ zero, even grounded predictions don't
encode per-step signal the policy can use.

---

## What Success Looks Like

Either experiment succeeding means:
1. Return > 18.38 (beats no-aug ceiling) — imagination is genuinely helping
2. Or at minimum: real accuracy >> zero accuracy in validate_awr — the policy is
   learning to use the embedding content, not just using it as a noise term

Currently neither condition has been achieved on PSF-style labels. Oracle labels get
condition 2 (real > zero in accuracy grid) but hurt at eval due to train/eval mismatch.
The two experiments probe orthogonal hypotheses about why.
