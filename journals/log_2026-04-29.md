**Critical finding**: even minimal edits to v2 (just adding one
paragraph or bullet) cause xxhighb's `enter_dungeon` rate to crash
from 40% → 7-20%. Return drops by 5-10 points. The policy is
**brittle to any prompt that diverges from the training distribution**,
even within the same overall structure.

This explains the "high content sensitivity but bad OOD steerability"
paradox:
- Δ(real-zero) = +0.7 (high content sensitivity) — policy reacts to
  prompt content
- BUT the reaction is mostly "this prompt is unfamiliar → behave badly"
  rather than "this prompt asks me to do X → do X".

The prompt-content steering pathway is more "stay on rails of training-
prompt distribution" than "follow new instructions in prompt".

## Codified benchmarks + scorecard (added 2026-04-29 ~22:50)

Two new tools enable rapid iteration on new variants:

- **`./tools/run_benchmark.sh <variant_tag> [num_episodes]`** — submits
  24-cell SLURM array (3 perf + 6 low-level steer + 15 high-level steer)
  via `slurm/jobs/full_benchmark.sh`. Wall ~50-90 min depending on slot
  availability.
- **`PYTHONPATH=. python tools/scorecard.py --variant <tag>`** — reads
  per-episode action lists, computes target metrics (action_DESCEND
  count, move_share_DOWN, length, eat_cow rate, ...), outputs markdown
  scorecard with WIN | NULL | WRONG-WAY verdicts (z>=1.0 threshold)
  and total WIN-rate.
- **`PYTHONPATH=. python tools/compare_variants.py v1 v2 v3`** —
  side-by-side multi-variant comparison.
- **`PYTHONPATH=. python tools/intrinsics_analysis.py --variant <tag>
  --cells <cell> [<cell>...]`** — replays per-cell episodes via Craftax
  env, accumulates HP/Food/Drink/Energy step-time histograms, prints
  comparison table.
