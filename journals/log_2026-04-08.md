
# 2026-04-08: Embedding Pathway Amplifies BC Toxicity

## Experiment Design

Three diagnostic runs using `train_awr_weighted_v2.py` with `ActorCriticAugLN` (512-width, 14.3M params), 100K steps. All use same data: 12.7M PPO training samples (126 files), 38K golden oracle samples.

| Experiment | hidden_mode | oracle_fraction | oracle_loss_weight | Regularization | Purpose |
|-----------|-------------|-----------------|-------------------|----------------|---------|
| Pure AWR (control) | zero | 0.0 | 0.0 | none (drop=0, wd=0, ent=0) | Sanity check v2 code |
| AWR+BC noaug | zero | 0.05 | 0.5 | drop=0.2, wd=1e-3, ent=0.03 | OOD golden BC without embeddings |
| AWR+BC aug | real | 0.05 | 0.5 | drop=0.2, wd=1e-3, ent=0.03 | Full BC+AWR with embeddings |

**Interpretation:** Raw norms are nearly identical (1.2% difference), but z-normalization amplifies per-dimension distributional differences. Oracle embeddings become 23.5% higher norm post-normalization, highly correlated (cosine 0.58), and have structured per-dimension offsets (dim_mean_spread 0.93 vs 0.02). The model can trivially identify oracle samples from their post-normalization embedding pattern.

## Union normalization experiment

**Hypothesis:** If separability is caused by normalization using training-only stats, computing stats over the union of training + oracle data should reduce separability and improve returns.

**Result:** Union norm had negligible effect on separability or training dynamics.

| Metric | Train-only norm | Union norm |
|--------|----------------|------------|
| Train post-norm L2 | 62.90 | 63.06 |
| Oracle post-norm L2 | 77.68 | 77.39 |
| Oracle dim_mean_spread | 0.926 | 0.921 |
| Within-oracle cosine | 0.579 | 0.576 |
| Final AWR loss | 2.14 | 2.15 |
| Final oracle loss | 0.07 | 0.07 |
| Final oracle entropy | 0.22 | 0.24 |
| Final train entropy | 2.13 | 2.02 |
| Final expl. variance | 0.91 | 0.93 |

**Why:** Oracle is only 38K samples vs 12.7M training — 0.3% of the union. Adding it barely shifts the global mean/std. The separability is inherent to the different game state distributions, not a normalization artifact.

## Conclusions

1. **OOD golden BC alone mildly degrades** (16.0→13.9, -13%) but doesn't collapse. High variance (σ=5.78) suggests the policy oscillates between good and bad modes.
2. **The embedding pathway makes BC catastrophic** (13.9→4.9, -65% from noaug; 16.0→4.9, -69% from control). With real embeddings, the model fully memorizes golden data (oracle loss 0.07) and partitions its policy using embedding-as-source-tag.
3. **Separability is inherent to the data**, not a normalization artifact — oracle and training embeddings come from fundamentally different game states (human floors 1-2 vs PPO floor 0), producing structurally different Qwen embeddings. Union normalization confirms this: including oracle in stats changes nothing.
4. **Regularization helps in the no-embedding case** — prior unregularized OW=0.5 runs collapsed harder. But regularization can't prevent collapse when the embedding gives a perfect source signal.
