## Landing: Track A (predonly) — full suite

### Online return (the *performance* axis)

| Policy | 50-ep mean ± std | Note |
|---|---|---|
| Track A predonly freezenone (real Gemini) | **18.98 ± 2.53** | this run |
| Apr-19 full-response freezenone | 18.82 ± 3.89 | Apr-19 winner |
| Unaug obs-only baseline | 18.38 ± 3.99 | floor |
| Track A predonly + constant-embed | 17.54 ± 3.77 | zero-content inside the same policy |

**Read**: predonly = full-response on *performance* (Δ=+0.16, within 1 SE).
Predonly is more *consistent* (σ 2.53 vs 3.89) which is a small qualitative
win. The 1.44 gap over constant-embed is the cleanest intra-policy content-use
signal we've measured — the hidden branch *is* doing something non-trivial.
But since real-Gemini = full-response, cleaner Prediction-only embeddings do
not lift raw return.

### Semantic suite (the *steerability* and *robustness* axes)

| Probe | Result | Axis |
|---|---|---|
| 50-ep die content (34/50 so far) | 18.48 ± 3.33 | *robustness* |
| 50-ep adversarial content (24/50 so far) | 18.73 ± 3.41 | *robustness* |
| Direction CF step-0 hidden-flip (50 eps) | **4.0%** | *steerability* |
| Multistep CF hidden-flip (115 records, 4 steps/ep × 30 eps) | **6.1% overall, 12% at step 300** | *steerability* |

- **Robustness is strong**: adversarial-/die-conditioned content does not
  meaningfully hurt return. The policy tolerates confabulated imagery.
- **Steerability is weak**: flipping the imagination (via 180°-rotated obs
  sent to Gemini) only changes the argmax movement in 4-12% of cases, vs
  40-84% when flipping the obs directly. Imagination moves the policy, but
  not directionally.
- **Horizon shape**: hidden-flip rate rises 3% → 7% → 3% → 12% from steps 0
  → 75 → 150 → 300. Not monotone, but the end of the horizon shows the
  clearest reading. More data (more eps × more intervention steps) would
  stabilize this curve.

### What this closes

The imagination branch in Track A:
- reads content (constant-embed < real),
- but reads it as an undifferentiated embedding-shape feature, not as a
  direction- / goal-specific forecast,
- produces tolerance to confabulation (robustness) but no steerability,
- is at parity with full-response on return (no performance lift from the
  predonly change).

This is the **policy-side bottleneck signature**. Fidelity could still add
something — that's what Track B/C tests — but if it plateaus, the architecture
/ training signal is the ceiling, not the prediction quality.

keeping straight:

  `Δ(real−zero)=+0.807`. Looks like memorization via the hidden branch — the
  oracle file was likely mixed into the training distribution.

  8 files): `acc_real=0.3993`, `Δ NLL(zero−real)=+0.194`, accuracy delta
  −2.67 pp. This is the credible number — small but nonzero.

Record: reporting the oracle-data Δ as "10× content-sensitivity" is a
memorization artifact. Use held-out PSF for validation going forward.

`feedback_in_vs_ood_semantic_tests.md` saved. The existing semantic suite
(direction CF, die/adv, HP-perturb) runs on fresh rollouts — that's the OOD
regime. We should additionally run each probe on *training-distribution*

from fundamental content-reading failure.

Planned script: `tools/in_distribution_semantic_probe.py` — loads (obs, hidden)
from `final_trajectories_psf_v2_cadence5_predonly_gemini_emb/` shards, does
forward passes with hidden swapped with another state's hidden (content CF)
and with obs flipped (obs CF). No Gemini calls needed. Outputs: obs-flip rate,
hidden-swap rate on N=500 offline states, per hidden-norm bucket.

This feeds directly into the *generalization* axis.
