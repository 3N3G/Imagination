# Apr 3–4: BC+AWR v3 & v4

## BC+AWR v3 Sweep (Apr 3)

First run with `train_awr_weighted_v2.py` (LayerNorm + Dropout architecture). Oracle head discriminates real vs zero-context hidden states.

4 variants, all with oracle_frac=0.1, oracle_weight=2.0, entropy=0.01:

| Variant | Dropout | Anneal | acc_real | acc_zero | Δ(real-zero) |
|---------|---------|--------|----------|----------|---------------|
| drop01 | 0.1 | No | 0.6187 | 0.4238 | **+0.1949** |
| drop01_anneal | 0.1 | Yes | 0.1813 | 0.2187 | -0.0374 |
| drop03 | 0.3 | No | 0.5634 | 0.4255 | +0.1379 |
| drop03_anneal | 0.3 | Yes | 0.1728 | 0.1949 | -0.0221 |

**Annealing kills the oracle head** — it reduces oracle weight to near zero over training, so the head never learns. No-anneal variants maintain positive discrimination.

### v3 Live Eval (100 episodes each)

| Variant | Return | Std |
|---------|--------|-----|
| drop01 | 6.04 | 4.74 |
| drop03 | 4.44 | 2.85 |

Very poor — oracle discrimination doesn't translate to gameplay. Possible causes:
- Oracle head discrimination is necessary but not sufficient
- Policy itself isn't conditioned well enough on the hidden states
- Need stronger oracle signal (higher weight, more oracle data)

## BC+AWR v4 Sweep (Apr 3–4, job 6944688)

Adjustments based on v3 failure: halved oracle fraction (0.05), added weight decay (1e-4), higher oracle weight options (0.5 vs 0.1).

| Model | ow_eff | Dropout | acc_real | acc_zero | Δ(real-zero) |
|-------|--------|---------|----------|----------|---------------|
| **bw05_drop03** | 0.50 | 0.3 | **0.7004** | 0.4562 | +0.2443 |
| **bw05_drop05** | 0.50 | 0.5 | 0.6987 | 0.4391 | **+0.2596** |
| bw01_drop03 | 0.10 | 0.3 | 0.6953 | 0.5515 | +0.1438 |
| bw01_drop05 | 0.10 | 0.5 | 0.6936 | 0.6426 | +0.0511 |

**Oracle weight = 0.50 clearly better.** Much stronger discrimination than v3. Low oracle weight (0.1) collapses, especially with high dropout.

Best validation accuracy (0.7004) and best discrimination (+0.2596) both from ow=0.50 models.

## Reflections

The BC+AWR approach is getting traction: v4 ow=0.5 models discriminate well (Δ~0.25 vs v3's Δ~0.19). The critical test is whether this translates to actual gameplay improvement. The v3→v4 improvements were:
1. Higher oracle weight (2.0 → 0.5 effective, but reduced oracle_frac from 0.1 to 0.05 so the batches are different)
2. Weight decay (0 → 1e-4) for better generalization
3. No annealing (confirmed kills the oracle head)
